En el proyecto de gestión de libros (Book Manager), vamos a construir un sistema backend que te permitirá manejar información sobre libros con diferentes funcionalidades y buenas prácticas modernas. A continuación te describo detalladamente qué vamos a hacer y qué aprenderás en cada componente:

Descripción general del proyecto Book Manager
Objetivo
Crear una API REST estructurada con arquitectura hexagonal donde los usuarios puedan crear, leer, actualizar, eliminar y buscar libros, gestionando múltiples campos como autor, género, año y ISBN. El sistema tendrá buena separación entre lógica de negocio, repositorios e interfaces, para facilitar mantenimiento y escalabilidad.

Qué haremos en cada capa del proyecto
1. Capa de dominio (domain)
Definiremos la entidad principal Book con campos clave: ID, título, autor, año, géneros, ISBN.

Crearemos interfaces/potentes contratos que describan las operaciones del repositorio (obtener todos, buscar por filtros, crear, actualizar, eliminar).

Aquí solo estará la lógica y contratos del negocio sin detalles específicos de implementación.

Aprenderás: Modelado de dominios, uso de interfaces en Go para desacoplar la lógica de negocio de la infraestructura.

2. Capa de aplicación (application)
Implementaremos servicios que usen las interfaces del dominio para realizar operaciones concretas.

Aquí llevaremos los casos de uso y la orquestación entre entidades y repositorios, aplicando patrones como Dependency Injection.

Validaciones de reglas de negocio (ejemplo: validar formato ISBN, campos obligatorios) pueden ir aquí.

Aprenderás: Separación clara de responsabilidades, encapsulación de lógica de negocio, tests unitarios centrados en servicios.

3. Capa de infraestructura (infrastructure)
Implementaremos la interfaz del repositorio usando una base de datos real (idealmente PostgreSQL o MySQL).

Incorporaremos consultas SQL complejas para búsqueda filtrada por autor, género y año.

Manejo de errores, conexiones eficientes, uso de migraciones y modelos optimizados.

Aprenderás: Interacciones con bases de datos relacionales en Go, prácticas eficientes para consultas, manejo correcto de errores DB.

4. Capa de API (api)
Crearás los endpoints HTTP para consumir los servicios de aplicación mediante Gin Framework.

Endpoints REST típicos:

GET /books (listado o búsqueda con query params)

GET /books/:id (detalle)

POST /books (crear libro)

PUT /books/:id (actualizar)

DELETE /books/:id (eliminar)

Implementarás validación de requests, manejo uniforme de errores y respuestas JSON bien estructuradas.

Aprenderás: Desarrollo de APIs REST con Gin, validación y manejo de errores, buenas prácticas en diseño de endpoints.

Funcionalidades y conceptos adicionales que se incorporarán
Búsqueda y filtros: soportar búsqueda por autor, género, año usando query parameters, usando SQL avanzados o funciones ORM.

Paginación y ordenamiento: permitir recibir parámetros para paginar resultados y ordenarlos, fundamental para API escalables.

Validaciones completas: campos obligatorios, formatos (ejemplo formato ISBN con regex, años validos).

Configuración con variables de entorno: usando godotenv para manejar configuraciones DB y entorno.

Tests automatizados: pruebas unitarias en servicios/aplicación y pruebas de integración para repositorios y capa API.

Contenerización (opcional): crear un Dockerfile para empaquetar la API para despliegue.

CI/CD básico (opcional): pipeline para correr pruebas y desplegar.

Qué aprenderás en resumen
Aplicar arquitectura hexagonal con separación estricta de capas

Desarrollar servicios backend robustos y mantenibles en Go

Construir APIs REST profesionales con Gin Framework

Trabajar con bases de datos SQL de forma avanzada para búsquedas y filtros

Validaciones y manejo de errores en APIs

Integrar buenas prácticas de desarrollo con pruebas y variables de entorno

Con esta base tendrás un proyecto backend profesional que podrás presentar en entrevistas y seguir escalando con características más avanzadas o infraestructura moderna como Kubernetes y microservicios.
